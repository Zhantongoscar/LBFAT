## 1. 产品定义与愿景

### 1.1 产品名称

**《工业逻辑竞技场》**
*Industrial Logic Arena*

### 1.2 产品定位

全球首款面向职业教育的**"虚实协同 + UGC 回合制对抗"工业排故推演沙盘平台**。

> 将枯燥的化工/制药参数分析，转化为一场硬核的**"团队资源管理（TRM）× 黑盒逻辑推理 × 回合制竞技"**三位一体实训工作站。

### 1.3 核心价值主张

| 维度 | 传统产品 | 工业逻辑竞技场 |
|------|----------|--------------|
| 学习动机 | 被动接受（要我学） | 主动竞争（我要赢） |
| 内容寿命 | 消耗型（玩一次即废） | 永久型（UGC无限生成） |
| 技能培养 | 单点操作（拧螺丝） | 系统思维（黑盒推理） |
| 硬件成本 | 高（专用3D实体模型） | 极低（通用感应桌垫） |
| 工程严谨性 | 中（手工预设剧本） | 高（Aspen/物理引擎校验）|

## 2. 市场痛点分析

### 2.1 职业教育当前痛点

**痛点一：传统仿真软件"点读机化"**
- 背答案式操作，缺乏空间感与动手参与感
- 上课极其枯燥，学生主动性极低

**痛点二：物理沙盘"盆景化"**
- 造价极其昂贵（几十至上百万人民币）
- 只能看、不能动，工艺一改、设备即废

**痛点三：单机剧本"复玩率极低"**
- 官方预设剧本有限，学生玩过一遍即知答案
- 设备迅速沦为吃灰摆设，投资回报率极低

**痛点四：缺乏"非技术技能"（Non-Technical Skills）训练**
- 真实工厂最需要的是极端压力下的**团队沟通**与**欺骗性仪表的甄别**能力
- 现有软件全在教"怎么拧螺丝"，忽视系统性判断和团队协作

### 2.2 目标用户

| 用户角色 | 描述 |
|----------|------|
| 主要用户 | 中高职化工、制药、石油化工、电力等专业学生 |
| 次要用户 | 指导教师（赛题出题方、评分方） |
| 扩展用户 | 企业新员工实训、技能鉴定考核机构 |

## 3. 核心产品形态

### 3.1 产品定义重构

**不是**"在沙盘上放故障看大屏爆炸"的演示工具；

**而是**——

> 💡 **大屏给出异常仪表数据，学生用物理积木在沙盘上进行侦探式搜证与排故的"工业逻辑对抗竞技场"**。

### 3.2 角色分工

```
┌─────────────────────────────────────────────────────────────────┐
```

## 4. 核心玩法机制

### 4.1 回合制对抗流程

```
回合一：布阵（甲方操作，乙方不可见）
  │
  ▼
回合二：侦查（乙方用探针积木逐一检测，甲方观战）
  │
  ▼
回合三：斩杀（乙方锁定根因，放置维修积木，系统裁判）
  │
  ▼
回合四：复盘（系统生成路径诊断报告，攻守互换）
```

### 4.2 甲方（出题方）机制

#### 4.2.1 出题方式：限制性组合（非自由编程）

甲方不能"瞎编"逻辑或违反物理规律。系统提供两种出题引擎（可选）：

**方案 A：因果逻辑链卡牌（基础版）**

系统内置经过专家验证的"原子化逻辑链"，例如：

| 链条编号 | 根因（故障源） | 传导路径 | 表象（大屏可见参数） | Cost |
|----------|--------------|----------|-------------------|------|
| C-001 | 冷却水管微堵 | 换热效率下降 | 反应釜温度T-102缓升 | 3点 |
| C-002 | 催化剂活性下降 | 反应速度变慢 | 产物纯度C-201下降 | 4点 |
| C-003 | 某阀门内漏 | 后段压力微升 | 压力P-305异常 | 3点 |
| C-004 | 传感器零点漂移 | 显示误差 | 仪表示数与实际不符 | 5点 |
| C-005 | 进料比失调 | 物料平衡破坏 | 多参数同步偏移 | 6点 |

甲方在**总破坏预算（如10点）**内，选择 1~3 条链条叠加，放置在沙盘物理节点上。

**方案 B：Aspen 仿真引擎（高级版，详见第6章）**

甲方直接在 Aspen 数字模型上施加干扰操作（如修改阀门开度、设备参数），Aspen 自动计算出稳态或瞬态偏移后的所有仪表参数，作为乙方看到的大屏数据。

#### 4.2.2 出题约束

| 约束类型 | 规则描述 |
|----------|----------|
| **物理合理性约束** | 只允许选择系统认可的故障类型（不允许"瞬间1000度"等超物理限制的故障） |
| **预算约束（Cost机制）** | 每回合总破坏预算有限，迫使甲方思考"最少预算布最深的坑" |
| **位置约束** | 故障卡牌只能放置在对应类型的工艺节点上（阀门类卡牌只能放管道节点） |
| **难度档次约束** | 根据场景难度设定，禁止使用高危快速故障（如设备突然断裂） |

### 4.3 乙方（解题方）机制

#### 4.3.1 探针预算制

乙方拥有有限的**探测预算（时间分/操作点数）**，不能穷举所有节点。

| 探测操作 | 实体积木 | 消耗预算 | 解锁信息 |
|----------|----------|----------|----------|
| 温度检测 | 红外测温积木 | 1点 | 该节点实时温度 |
| 压力检测 | 压力表积木 | 2点 | 该节点实时压力 |
| 流量检测 | 流量计积木 | 2点 | 该节点流量值 |
| 采样分析 | 分析仪积木 | 3点 | 成分/浓度数据 |
| 拆卸维修 | 扳手/修复积木 | 5点 | 直接修复（若选错则判负） |

#### 4.3.2 团队角色分工（TRM 强制协作）

| 角色 | 位置 | 职责 |
|------|------|------|
| **中控员** | 大屏前 | 观察大屏综合参数，进行逻辑推断，指挥巡检员 |
| **巡检员** | 沙盘旁 | 执行探针操作，将实体积木放置到指定节点 |

两个角色视角分离，**必须高频沟通才能高效破局**，模拟真实工厂的中控室与现场操作员配合场景。

### 4.4 胜负判定

| 结果 | 条件 | 得分方 |
|------|------|--------|
| **乙方完胜** | 在预算耗尽前，准确找到所有故障根因并修复 | 乙方得学分 |
| **甲方完胜** | 乙方预算耗尽前未找到根因 OR 误诊导致系统恶化 | 甲方得学分 |
| **平局** | 乙方找到部分故障，预算恰好耗尽 | 按比例计分 |

### 4.5 攻守互换

每局结束后，甲乙方交换角色，进入下一轮。这迫使每位学生都必须掌握"出题思维"（如何布置最隐蔽的故障）和"解题思维"（如何最高效地定位根因），双重提升。

## 6. Aspen 仿真引擎集成方案（最新）

> 💡 **核心思路革新**：放弃手工编写"因果逻辑链"（成本高、难以穷举），改用 **Aspen 化工过程模拟软件作为仿真核心**，在真实化工模型上进行干扰操作，让 Aspen 自动计算出故障状态下的所有仪表参数。

### 6.1 方案核心优势

| 维度 | 手工因果逻辑链（旧方案）| Aspen 仿真引擎（新方案）|
|------|----------------------|----------------------|
| 工程准确性 | 依赖专家手工归纳，可能有盲区 | **基于严格热力学/流体力学计算，100% 物理准确** |
| 内容可扩展性 | 需人工不断增补链条 | **Aspen 模型可覆盖任意化工工艺场景** |
| 出题难度（甲方）| 需理解抽象逻辑链 | **直接操作熟悉的工艺参数，更直觉化** |
| 计算实时性 | 即时（查表式） | 需要 Aspen 稳态/动态计算（秒级到分钟级）|
| 部署成本 | 低 | **需要 Aspen 软件授权** |

### 6.2 Aspen 集成工作原理

```
甲方操作（造局）
    │
    ▼
在 Aspen 数字孪生模型中施加干扰
（如：修改某阀门 Cv 值 → 模拟内漏；
    调整换热器传热面积 → 模拟结垢堵塞；
    修改反应动力学参数 → 模拟催化剂失活）
    │
    ▼
Aspen 自动执行稳态/动态计算
（计算所有流股温度、压力、流量、组分的偏移值）
    │
    ▼
计算结果导出至"大屏数据总线"
（只向乙方展示仪表参数，隐藏根因）
    │
    ▼
乙方在实体沙盘上进行探针操作
（每次探针结果 = 对应工艺节点的 Aspen 计算值）
    │
    ▼
乙方提交修复方案 → Aspen 验证修复正确性
```

### 6.3 甲方出题的约束规则（基于 Aspen）

为了保证"可玩性"和"教学价值"，即使使用 Aspen 引擎，仍需要规则约束：

#### 6.3.1 干扰操作类型限制（难度分级）

| 难度等级 | 允许的干扰操作类型 | 禁止的操作 |
|----------|------------------|------------|
| **初级** | 传感器零点漂移、阀门开度微调（±10%以内）| 设备突然爆炸、断裂等极端物理破坏 |
| **中级** | 换热器效率下降、催化剂活性降低 | 超过工艺设计极限的参数设定 |
| **高级** | 多点并发故障叠加（受 Cost 限制） | 无法在真实操作中实现的操作 |
| **竞技级** | 包含误导性传感器故障（双重欺骗） | —— |

#### 6.3.2 扰动幅度约束（防止"秒杀型"题目）

甲方设置的干扰操作必须满足：
- **Aspen 仍能收敛求解**（保证物理上合理）
- **故障传导到可测点的参数偏差不超过设定阈值**（防止一目了然）
- **故障不会导致系统在1分钟内崩溃**（保证乙方有足够的排查时间）

#### 6.3.3 提示系统（分难度显示线索）

| 难度设定 | 大屏提示级别 |
|----------|------------|
| 教学模式 | 高亮显示异常参数区域、给出排查建议方向 |
| 标准模式 | 仅显示参数偏差，不给出提示 |
| 竞技模式 | 可设置"噪声干扰"（部分正常传感器显示随机扰动） |

### 6.5 MVP 阶段的 Aspen 集成策略

考虑到 Aspen 软件许可证成本和集成周期，建议分阶段实施：

| 阶段 | 仿真引擎 | 说明 |
|------|----------|------|
| **MVP（第一阶段）** | 手工因果逻辑链卡牌 | 快速验证玩法核心，3个月内可上线 |
| **V1.1（第二阶段）** | 引入 Aspen 稳态模型 | 针对 1~2 个标准工艺（如精馏+反应） |
| **V2.0（第三阶段）** | Aspen 动态模型（全场景） | 支持任意工艺，真正的"工程级仿真竞技" |

## 7. 硬件方案

### 7.1 极简硬件原则

**核心理念**：硬件是可替换的底座，软件和逻辑引擎才是护城河。

### 7.2 硬件组成

```
┌────────────────────────────────────────────────────────────┐
│                       工作站套件                             │
│                                                             │
│  ① 大屏显示器（≥55寸，4K）     ② 甲方平板/电脑（出题端）     │
│     └─ DCS 中控模拟仪表盘          └─ 故障部署界面           │
│                                                             │
│  ③ 智能感应工作桌                                           │
│     ├─ NFC 矩阵感应层（A2纸大小即可）                         │
│     └─ 可替换 2D 工艺流程底图纸（打印即用）                    │
│                                                             │
│  ④ 探针积木套件（通用，约10~15个）                            │
│     ├─ 温度探针积木（红色）                                  │
│     ├─ 压力探针积木（蓝色）                                  │
│     ├─ 流量探针积木（绿色）                                  │
│     ├─ 分析仪积木（黄色）                                    │
│     └─ 维修积木（扳手形状）                                  │
└────────────────────────────────────────────────────────────┘
```

### 7.3 感知方案对比

| 方案 | 技术路线 | 精度 | 成本 | 推荐阶段 |
|------|----------|------|------|---------|
| **NFC 矩阵** | 桌面嵌入 NFC 读卡器阵列 | 高 | 中 | V1.0 |
| **视觉识别** | 桌面顶置摄像头 + ArUco/QR码积木 | 中 | 低 | MVP |
| **蓝牙定位** | 积木内置 BLE Beacon | 高 | 高 | V2.0 |

### 7.4 硬件复用性（一套硬件，无限工艺）

只需更换"2D 工艺底图纸"（打印成本约1元/张），即可支持：
- 化工精馏工艺
- 化工反应工艺
- 制药发酵工艺
- 电力变电工艺
- 石油炼化工艺

## 9. 竞品与差异化分析

| 竞品类型 | 代表产品 | 核心缺陷 | 本产品优势 |
|----------|----------|----------|----------|
| 化工仿真软件 | Aspen EDR, HYSYS | 单机操作，无对抗，无协作 | UGC回合制对抗，无限复玩 |
| 3D 物理沙盘 | 传统模型沙盘 | 造价高，内容固定，不能互动 | 极简硬件，可替换工艺图纸 |
| VR 实训系统 | 各类 VR 化工模拟 | 单人体验，硬件昂贵，易晕 | 实体触觉交互，团队协作 |
| 逃脱房/剧本杀 | 工业主题逃脱房 | 与专业知识脱钩，娱乐性 | 严格对标职教课程标准 |

**核心差异化（护城河）**：
1. **UGC + 回合制** = 内容永不枯竭（竞品无）
2. **Aspen 仿真验证** = 工程准确性无可挑剔（竞品无）
3. **实体积木 + 感应桌垫** = 触觉沉浸 × 极低硬件成本（竞品无）

## 10. 商业模式

### 10.1 产品分层

| 版本 | 定价策略 | 适用场景 |
|------|----------|---------|
| **基础版** | 硬件一次性买断 + 基础软件 | 中职院校基础实训课 |
| **专业版** | 硬件 + SaaS年费（含Aspen仿真接口）| 高职/应用型本科 |
| **竞技版** | 联赛专用，按赛季收费 | 大赛、职教杯、技能鉴定 |

### 10.2 收入来源

- 硬件销售（桌垫+积木套件）
- SaaS 软件订阅（逻辑引擎+题库+报告系统）
- 内容授权（行业工艺模型包）
- 赛事服务（全国职教竞技联赛冠名/承办）
- 企业定制（新员工培训版）

### 10.3 盈利模型亮点

由于采用 UGC 模式，**学生自己就是免费的关卡设计师**，大幅降低内容运营成本；SaaS 化的软件引擎使硬件仅作为"底座销售"，真正的竞争壁垒和持续收益来自软件生态。

## 12. 风险与应对策略

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| **Aspen 软件许可证成本高** | 提高客单价，限制普及 | MVP 阶段先用因果链卡牌；与 Aspen 官方洽谈教育授权折扣 |
| **学生出题质量参差不齐（UGC）** | 劣质题目损害体验 | 建立教师审核机制；引入题目评分系统（好题多次被选用则加分给出题者） |
| **硬件感知精度不稳定** | 积木识别出错，影响体验 | MVP 阶段优先视觉识别方案；正式版切换 NFC 矩阵 |
| **工艺底图版权问题** | 使用真实企业工艺图纸的法律风险 | 由化工专业教师开发"教学用标准工艺模型"，不使用真实工厂图纸 |
| **传统教师接受度低** | 老师不愿配合推广 | 主打"自动出考卷+自动评分"的减负卖点；组织教师培训和体验赛 |

## 附录：名词解释

| 术语 | 解释 |
|------|------|
| **TRM** | Team Resource Management，团队资源管理，源自民航CRM理念，强调团队在极端情况下的沟通与协作 |
| **UGC** | User Generated Content，用户生成内容，指学生自行创作关卡/题目 |
| **PVP** | Player vs Player，玩家对战玩家（区别于 PVE 人机对战）|
| **DCS** | Distributed Control System，分布式控制系统，化工厂的中央控制系统 |
| **Aspen HYSYS** | 一款工业级化工稳态/动态过程模拟软件，广泛用于化工设计和操作培训 |
| **Cost** | 游戏机制中的"资源点数"，限制甲方每回合的破坏预算 |
| **因果逻辑链** | 工程专家手工编写的"根因→传导→表象"三段式逻辑单元，是基础版的核心内容 |
| **黑盒推理** | 面对系统外部可观测的异常（仪表数据），逆向推断内部根因的思维模式 |
